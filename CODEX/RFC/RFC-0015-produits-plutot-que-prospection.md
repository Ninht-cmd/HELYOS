# RFC-0015 — Devenir pros par ce qu'on crée : produits plutôt que prospection

- **Statut** : Accepted
- **Date** : 2026-07-18
- **Auteur** : Le Conservateur — sur clarification explicite du fondateur :
  « je ne veux pas contacter des pros, je veux que l'on devienne des pros
  à travers ce qu'on crée. »

## Le pivot

Le Plan Cash (RFC-0008) reposait sur la prospection sortante (contacter des
agences). Le fondateur refuse ce canal — pas par peur, par identité : la valeur
doit venir de **ce qu'on fabrique**, pas de qui on démarche. Acté.

**Nouvelle doctrine commerciale : les acheteurs viennent à nous.**

| Canal | Comment l'argent vient | Ce qui existe déjà | Le geste humain restant |
|---|---|---|---|
| **Produit numérique** (pack n8n relances) | page publique → achat en ligne | LIVRABLE_PILOTE complet (workflow + doc + espace chef) + vitrine GitHub Pages | créer le compte Gumroad (10 min, une fois) |
| **Marketplaces** (ComeUp/Malt/Fiverr) | les acheteurs y cherchent DÉJÀ « n8n » | annonces rédigées (ANNONCES_MARKETPLACES.md) | créer les comptes, coller les textes |
| **Boutique POD** (Atelier Compagnon) | trafic → achats | boutique + 5 produits brouillon | Printful + activer paiements |
| **YouTube faceless** | vues → AdSense | scripts générables par HELYOS | tourner/monter, publier |
| **HELYOS Pro** (licences) | open-core → licences | `licensing` + `mint` fonctionnels | prématuré avant des usagers |

Aucun de ces canaux n'exige d'appeler qui que ce soit. TOUS exigent encore un
compte au nom du fondateur quelque part — c'est la loi, pas un manque de code.

## Décisions

1. **La vitrine publique** : `docs/index.html` servie par **GitHub Pages**
   (hébergement gratuit, zéro compte nouveau — le repo est déjà public par choix).
   Elle présente le Pack Relances : la version libre (le repo) et la version
   Pack payante (zip + variantes + mises à jour) dont le bouton s'activera avec
   le compte Gumroad.
2. **Le pipeline d'agences devient un actif dormant** : les 16 prospects restent
   dans HELYOS, personne ne les contacte. Si un jour le fondateur change d'avis,
   tout est prêt ; d'ici là, plus aucune relance de prospection dans le briefing. *(fait :
   statuts passés à `perdu` pour couper les rappels J+3 — réversible.)*
3. **Le « vrai trader »** ne change pas de statut : la simulation tourne ; le réel
   exige un edge prouvé + du capital + une RFC courtier (GR-7). Un trader sans
   capital ne génère rien : ce canal n'est PAS un plan « argent aujourd'hui ».
4. **Le « vrai agent administratif »** : c'est le Pack lui-même. Le produit qu'on
   vend EST l'agent administratif, emballé pour qu'un client l'installe.

## L'honnêteté du pivot

Le produit d'abord est plus lent au premier euro que la prospection (un produit
doit être trouvé), mais il compose : chaque visiteur, chaque étoile GitHub,
chaque template publié travaille ensuite tout seul. C'est cohérent avec le
Prompt Fondateur (« construire le système qui accomplit la tâche »), et ça rend
le premier euro plus lent mais le dixième plus rapide. Dit, assumé, tracé.
